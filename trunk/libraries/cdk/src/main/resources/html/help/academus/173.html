View All Active Offerings : Managing Offerings

 

Offering States

There are two types of Offerings, active and inactive. When an Offering is created, it is considered Active. When an Active Offering is deleted, its status becomes Inactive. Most people cannot view Inactive Offerings. This is a tidy way of storing unused Offerings for future use, without them cluttering up the Active Offerings list. This feature also allows sponsors to continue to have access to an offering after the conclusion of a term, while removing the offering from the students' or members' view.


View Active Offerings

To view Active Offerings a user has two options. Users with proper privileges can view all offerings by searching within the Offering Administration channel. The Offering Subscription channel also allows users to view offerings with Open and Request/Approve enrollment types. To view offerings within the Offering Subscription channel, follow these steps:

  1. From the Navigation channel, click the Offering Subscription link.
  2. Enter search criteria for the offerings you wish to view, or leave all fields blank to view all offerings.
  3. Click Search, you will now see as many Offerings as you have set for each page.
  4. To more offerings on one page, locate the Per Page drop-down menu, and select the desired number.
  5. Now click the Edit icon next to this menu.
  6. You should now see all Offerings on one page.

 

Adding a New Offering

To add a new Offering, complete these steps:

  1. Click the Add icon at the top of the Offering Administration channel.
  2. You should now see the Add Offering screen.
  3. Choose a Topic.
  4. Enter an Offering Name and a short Description in the text fields.
  5. Select a User Enrollment Model based upon the way users can be enrolled. Open Enrollment is open for any users to enroll. Request/Approve enrollment requires the user to get approval from the Offering Sponsor before they will be enrolled. In the Facilitator model, users can only be enrolled by the Offering Sponsor or a system administrator; they don't have the means to request enrollment. An enrollment model of SIS is used for integrations with third party Student Administration Systems.
  6. Select an Enrolled User Default Type from the drop-down list.
  7. Enter any additional details about the offering such as Offering ID, Term, Start Date, End Date, Meeting Days, Meeting Times, and Location.
  8. When you are satisfied with all settings for this offering, click the Submit button.
  9. You will now be able to search for and view your new offering in the Offering Administration channel.


Viewing Details of an Offering

To view the details of the Offering you just added, or for any Active Offering, follow these steps:

  1. In the Navigation channel, click the Administration link.
  2. Now locate the Offering Administration channel and click Search.
  3. Enter the Name of the Offering to be located.
  4. Click on the Name of the Offering or the magnifying glass icon next to it.
  5. You will now be able to see the Offering Details screen.
